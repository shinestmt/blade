Blade

blade使用指南

首先你搭建好了blade框架的基础环境，那么就开始入手怎么写一个简单的web程序了！

这里给大家理一下流程和后续你要做的

  1. 以maven or 普通方式引入blade的jar包
  2. 配置web.xml并建立包和App.java文件
  3. 编写App.java需要的基础环境
  4. 编写路由和逻辑代码

现在是第三步了，前面已经完成了2步，那么下面为大家剖析一下App.java这个类要怎么写。

约定优于配置的原则很多人都懂，blade里面就是这么做的，原因很简单，这么做高效，快速！

public class App extends BladeApplication{

    Logger logger = Logger.getLogger(App.class);

    @Override
    public void init() {

        // 设置路由、拦截器包所在包
        Blade.defaultRoute("blade.sample");

        // 设置视图目录和视图后缀
        Blade.view("/WEB-INF/views/", ".jsp");

        // 设置静态文件目录
        Blade.staticFolder("/static");

    }

    public static void main(String[] args) {
        Blade.run(App.class, 9000);
    }
}

这是一个简单的全局初始化配置，先了解这个再继续复杂的操作，

  • 第一步设置路由、拦截器包所在包 Blade.defaultRoute("blade.sample")方法传入你的基础包，我们一般的包命名都是 com.xxx.aaa,com.xxx.bbb，那么blade内部默认所有的路由都放在blade.sample.route包下， 所有的拦截器都放在blade.sample.interceptor包下。当然，它提供单独配置，这里就体现了约定优于配置，我知道这个规则就一行代码搞定， 如果单独配置可以酱紫：

      Blade.routes("blade.sample.route");
  Blade.interceptor("blade.sample.interceptor");

    其实是一样的！我提供接口，你随意。

    路由配置支持多个包配置，支持递归配置

      // 多个包
  Blade.routes("blade.sample.route", "blade.sample.admin.route");

  // 递归
  Blade.routes("blade.sample.route.*");

  • 第二步设置视图目录和视图后缀

    这就有点儿类似spring中的视图解析器配置，那个是xml格式的。 Blade.view("/WEB-INF/views/", ".jsp");这里我们指定了所有的视图文件都在/WEB-INF/views/下，所有的视图后缀是.jsp的。

  • 第三步设置静态文件目录

    我们开发web程序都会有专门存放js,css等资源文件的目录，那这些文件是不用交给blade的handler解析的， 所以配置一个或多个静态目录把它过滤掉就O了。 Blade.staticFolder("/static");这句话是指web应用下以/static开头的文件不进行处理，由web容器处理，如:http://127.0.0.1:9000/static/img/hello.jpg 不用去找路由了，直接由web容器找到资源文件返回即可。

如果你使用jetty，blade内嵌了jetty服务器，一般调试用即可。 Blade.run(App.class, 9000); 设置一个启动配置类和服务端口，启动后访问http://127.0.0.1:9000即可
