依赖注入

依赖

如果在 Class A 中，有 Class B 的实例，则称 Class A 对 Class B 有一个依赖。

例如下面类 Human 中用到一个 Father 对象，我们就说类 Hello 对类 User 有一个依赖。

public class Hello {

    User user;

    public Hello() {
        user = new User();
    }
}

仔细看这段代码我们会发现存在一些问题：

(1) 如果现在要改变 user 生成方式，如需要用new User(String name)初始化 user，需要修改 Hello 代码

(2) 如果想测试不同 User 对象对 Hello 的影响很困难，因为 user 的初始化被写死在了 Hello 的构造函数中

(3) 如果new User()过程非常缓慢，单测时我们希望用已经初始化好的 user 对象 Mock 掉这个过程也很困难

依赖注入

最新一代容器称为轻量级容器，它们使用一个共同设计原理：依赖注入。

对这个简单思想来说，这是一个复杂的术语。依赖注入让您将对象和它所依赖的东西交给第三方。

然后第三方创建所有对象并将它们绑在一起。

上面将依赖在构造函数中直接初始化是一种 Hard init 方式，弊端在于两个类不够独立，不方便测试。

我们还有另外一种 Init 方式，如下：

public class Hello {

    User user;

    public Hello(User user) {
        this.user = user;
    }
}

上面代码中，我们将 user 对象作为构造函数的一个参数传入。

在调用 Hello 的构造方法之前外部就已经初始化好了 User 对象。

像这种非自己主动初始化依赖，而通过外部来传入依赖的方式，我们就称为依赖注入。

现在我们发现上面 1 中存在的两个问题都很好解决了，简单的说依赖注入主要有两个好处：

(1) 解耦，将依赖之间解耦。

(2) 因为已经解耦，所以方便做单元测试，尤其是 Mock 测试。

Blade 中的依赖注入

  • 配置要进行依赖注入的包
  • 对要用IOC管理的类进行配置
  • 使用@Inject注解进行注入

配置：

// 配置IOC容器要管理的包
Blade.ioc("blade.sample.service.*");

依赖注入的实现有多种途径，而在 Java 中，使用注解是最常用的。

通过在字段的声明前添加 @Inject 注解进行标记，来实现依赖对象的自动注入。

public class Hello {

    @Inject
    User user;

    public Hello() {
    }
}

上面这段代码看起来很神奇：只是增加了一个注解，User 对象就能自动注入了？

这个注入过程是怎么完成的？

已经有很多框架完成了这个功能，诸如struts、spring等主流框架，

在blade的内核中也集成了简易的依赖注入(IOC)容器，让对象管理更加简单。

Blade的IOC采用注解方式配置，无须xml，下面是一个使用Blade IOC的例子

public class UserRoute {

    @Inject
    UserService userService;

    public void run(){
        userService.sayHello();
    }
}

@Component
public class UserService {

    public void sayHello() {
        System.out.println("userservice say hello");
    }

}

我们用@Inject注解在UserRoute对象中注入了UserService的一个实例，UserService是一个类，

在类上面标注了@Component注解来表示这是一个需要注入的IOC对象。那么接口的情况呢？

public class UserRoute {

    @Inject
    UserService userService;

    public void run(){
        userService.sayHello();
    }
}

public interface UserService {

    public void sayHello();

}

@Component
public class UserServiceImpl implements UserService{

    @Override
    public void sayHello() {
        System.out.println("userservice say hello");
    }

}

它应该是这样的，同样你定义一个UserService的接口，

在它的实现类上标注一个@Component注解就代表该接口的实例对象是UserServiceImpl，

你要确保一个接口没有多个实现类或一个接口只能注入一个实现类，否则无法成功注入。
